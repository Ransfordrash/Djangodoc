Introduction à Django

Django est un framework web open-source basé sur Python, conçu pour faciliter la création de sites web robustes, rapides et évolutifs. Il repose sur le modèle MTV (Modèle-Template-Vue), une variante du modèle MVC. Ses principales caractéristiques incluent :

  • Rapidité de développement :

    Django est livré avec de nombreux outils intégrés (authentification, gestion de formulaires, etc.), ce qui permet de gagner du temps.

  • Sécurité :

    Il protège contre les attaques courantes comme les injections SQL, les failles XSS et CSRF.

  • Scalabilité :

    Il convient aussi bien à de petites applications qu'à des plateformes complexes.

Installer et configuration

Pour utiliser Django, suivez ces étapes :

  1. Installer Django
    2. pip install django
  2. Créer un projet Django :
    3. django-admin startproject nom_du_projet

    Cette commande crée une structure de base avec des dossiers comme settings, urls, et wsgi.

  3. Démarrer le serveur de développement :
    4. python manage.py runserver
  4. Créer une application :

    Les projets Django sont divisés en applications. Pour en créer une :

    5. python manage.py startapp nom_de_l_application
Structure de base d'un projet Django

Un projet Django suit une structure bien définie :

  • manage.py :

    Un script qui permet d’exécuter des commandes liées au projet.

  • settings.py :

    Fichier de configuration où se trouvent les paramètres du projet (base de données, applications, sécurité, etc.).

  • urls.py :

    Définit les URL qui mappent les vues.

  • models.py :

    Contient les modèles définissant la structure de la base de données.

  • views.py :

    Contient la logique pour traiter les requêtes HTTP et retourner les réponses.

  • templates :

    Dossier où sont stockés les fichiers HTML.

Principal fonctionnalités de Django

Django offre un large éventail de fonctionnalités pour faciliter le développement web :

  • ORM (Object-Relational Mapping) :

    Interaction avec des bases de données SQL sans écrire de requêtes SQL. Exemple :

    • from django.db import models                                                          class Article(models.Model):                      titre = models.CharField(max_length=100)            contenu = models.TextField()                      date_publication = models.DateTimeField(auto_now_add=True)

    Avec ce modèle, Django génère automatiquement les tables correspondantes dans la base de données.

  • Système d’authentification :

    Gestion des utilisateurs (connexion, inscription, permissions).

  • Administration automatique :

    Une interface admin générée automatiquement. Activez-la en ajoutant admin dans INSTALLED_APPS

  • Middleware :

    Gestion des requêtes et des réponses HTTP via des couches intermédiaires.

  • Support multilingue :

    Outils intégrés pour traduire les applications.
